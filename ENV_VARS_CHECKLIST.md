## Quick Reference

## ✅ Required Variables (Must Have)

These variables are **required** for the application to work:

### Database (Supabase/PostgreSQL)
```bash
DATABASE_HOST=                    # e.g., aws-0-us-east-1.pooler.supabase.com
DATABASE_PORT=5432
DATABASE_NAME=postgres
DATABASE_USER=                    # e.g., postgres.xxxxxxxxxxxxxxxxxxxx
DATABASE_PASSWORD=                # Generated by Supabase
DATABASE_SSL=true                 # Must be true for Supabase
```
**Setup Guide**: See [MVP.md Phase 1](./MVP.md#phase-1-database--supabase-postgresql-free)

---

### JWT Authentication
```bash
JWT_SECRET=                       # Generate: openssl rand -base64 64
REFRESH_TOKEN_SECRET=             # Generate: openssl rand -base64 64
JWT_EXPIRES_IN=15m
REFRESH_TOKEN_EXPIRES_IN=30d
```
**Generate Secrets**:
```bash
openssl rand -base64 64
openssl rand -base64 64
```

---

### Server Configuration
```bash
PORT=3000
NODE_ENV=production
CORS_ORIGIN=                      # e.g., https://your-app.vercel.app
FRONTEND_URL=                     # e.g., https://your-app.vercel.app
```

---

### Image Storage (Cloudinary)
```bash
CLOUDINARY_CLOUD_NAME=            # From Cloudinary dashboard
CLOUDINARY_API_KEY=               # From Cloudinary dashboard
CLOUDINARY_API_SECRET=            # From Cloudinary dashboard (click "Reveal")
```
**Setup Guide**: See [MVP.md Phase 2](./MVP.md#phase-2-image-hosting--cloudinary-free)

## 📋 Platform-Specific Setup

### For Render.com (Backend)

1. Go to: https://dashboard.render.com/
2. Select your backend service
3. Click **"Environment"** (left sidebar)
4. Click **"Add Environment Variable"** for each variable above
5. Click **"Save Changes"** (triggers auto-redeploy)

**Required Variables**:
- All "Required Variables" from above
- INSTAGRAM_*, FACEBOOK_*, TIKTOK_* (if using social media integration)

---

### For Vercel (Frontend)

1. Go to: https://vercel.com/
2. Select your project
3. Go to **Settings** → **Environment Variables**
4. Add these variables:

```bash
NEXT_PUBLIC_API_URL=              # e.g., https://your-backend.onrender.com/v1
NEXT_PUBLIC_APP_URL=              # e.g., https://your-app.vercel.app
```

## 🔍 Quick Validation Checklist

Before deploying, verify:

### Database Connection
- [ ] DATABASE_HOST is correct (no typos)
- [ ] DATABASE_PASSWORD matches Supabase password
- [ ] DATABASE_SSL=true is set
- [ ] Can connect via `psql` command

### Security
- [ ] JWT_SECRET is different from development
- [ ] REFRESH_TOKEN_SECRET is different from JWT_SECRET
- [ ] Secrets are long and complex (64+ characters)
- [ ] No secrets committed to Git

### URLs & CORS
- [ ] CORS_ORIGIN matches frontend URL exactly (no trailing slash)
- [ ] FRONTEND_URL matches Vercel deployment URL
- [ ] All OAuth redirect URIs point to correct backend URL

### Image Storage
- [ ] CLOUDINARY_CLOUD_NAME is correct
- [ ] CLOUDINARY_API_SECRET is revealed and copied correctly
- [ ] Can upload test image via API

### OAuth (if enabled)
- [ ] All three Instagram variables set
- [ ] All three Facebook variables set
- [ ] All three TikTok variables set
- [ ] Redirect URIs configured in each platform's developer console
- [ ] Apps are in correct mode (Development/Live)

## 🚨 Common Mistakes

### 1. Wrong Database SSL Setting
```bash
❌ DATABASE_SSL=false               # Will fail to connect to Supabase
✅ DATABASE_SSL=true                # Correct for Supabase
```

### 2. CORS URL Mismatch
```bash
❌ CORS_ORIGIN=http://your-app.vercel.app     # Wrong protocol
❌ CORS_ORIGIN=https://your-app.vercel.app/   # Trailing slash
✅ CORS_ORIGIN=https://your-app.vercel.app    # Correct
```

### 3. OAuth Redirect URI Mismatch
```bash
❌ INSTAGRAM_REDIRECT_URI=http://localhost:3000/v1/oauth/instagram/callback     # Local URL in production
✅ INSTAGRAM_REDIRECT_URI=https://your-backend.onrender.com/v1/oauth/instagram/callback
```

### 4. Missing /v1 in API URLs
```bash
❌ NEXT_PUBLIC_API_URL=https://your-backend.onrender.com       # Missing /v1
✅ NEXT_PUBLIC_API_URL=https://your-backend.onrender.com/v1    # Correct
```

### 5. Weak JWT Secrets
```bash
❌ JWT_SECRET=secret                          # Too weak
❌ JWT_SECRET=my-secret-key                   # Too weak
✅ JWT_SECRET=xK9mP2nQ5rT8wE3vY7zA...         # Strong (64+ chars)
```

## 🔐 Security Best Practices

1. **Never commit .env files to Git**
   ```bash
   # Verify .env is in .gitignore
   cat .gitignore | grep .env
   ```

2. **Use different secrets for development and production**
   - Generate new JWT secrets for production
   - Use separate OAuth apps for dev/prod

3. **Rotate secrets periodically**
   - Change JWT secrets every 6 months
   - Regenerate OAuth secrets annually

4. **Limit access to production environment variables**
   - Only give team members access who need it
   - Use Render/Vercel team features for access control

5. **Monitor for leaked secrets**
   - Use GitHub secret scanning
   - Check for accidental commits
